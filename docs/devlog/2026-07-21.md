# 2026-07-21

## 상황

Phase 2 테스트 환경 구축. Vitest + RTL, Playwright를 세팅하고, 슬러그 생성 함수(`generateSlug`)를 TDD로 처음 만들어봄.

## 막힌 지점 (에러 원문)

`pnpm run test:watch`(= `vitest`)를 실행했는데 watch 모드로 안 걸리고 한 번 실행하고 바로 끝남. `pnpm test`(= `vitest run`)랑 결과가 똑같아 보여서 스크립트가 잘못된 줄 알았음.

## 원인

Vitest는 실행 환경이 진짜 인터랙티브 터미널(TTY)이 아니라고 판단하면 watch 모드로 걸어도 자동으로 1회 실행 후 종료로 전환한다. Claude Code의 `!` 셸 연동처럼 출력을 캡처해서 보여줘야 하는 환경은 TTY가 아니라서 이 케이스에 해당했음.

## 해결

별도 터미널 앱(Terminal/iTerm)을 직접 열어서 `pnpm run test:watch`를 돌리면 정상적으로 watch 모드가 유지되고 파일 저장 시 자동 재실행됨을 확인.

## 배운 점

- TDD 사이클(Red → Green → Refactor)을 실제로 도는 느낌: 테스트를 먼저 쓰고 실패를 본 다음, 최소 구현으로 통과시키고, 통과 상태를 유지하면서 정리하는 리듬이 생각보다 명확했음.
- 모든 새 테스트가 항상 Red로 시작하는 건 아님 — 이미 구현이 커버하는 케이스를 테스트로 추가하면 바로 Green으로 통과하기도 함 (이 경우도 회귀 방지용 문서화로서 가치가 있음).
- 슬러그처럼 한글 제목을 지원해야 하는 함수는 정규식에서 `\p{L}\p{N}`(유니코드 문자/숫자 클래스)를 써야 ASCII 외 문자가 안 날아감.
- CLI 도구(Vitest 등)가 "터미널이 인터랙티브인지"를 감지해서 동작을 바꾸는 경우가 있다는 걸 처음 알게 됨 — CI 환경에서 watch 모드로 매달리는 걸 막기 위한 안전장치.
